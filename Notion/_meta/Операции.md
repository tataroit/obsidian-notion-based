# Операции

> Этот файл определяет доступные операции и сценарии работы с базой знаний.
> При старте сессии (фраза "давай начнём работу" или аналогичная) — показать меню ниже и ждать выбора.

---

## Меню операций

При старте показывай это меню и жди выбора:

---

Еженедельные
1. `Еженедельный обзор` — Входящие + статусы доменов + фокус + рефлексия
2. `Рефлексия` — заполнить рефлексию вопрос за вопросом

Атомарные
1. `Новая база знаний` — создать knowledge-файл по новой теме
2. `Углубить базу знаний` — добавить материал в существующий файл
3. `Разобрать Входящие` — разобрать мысли из блока Входящие в Home.md

---

После выбора — выполняй операцию по инструкции ниже.

---

## Операция 1 — Еженедельный обзор

**Результат:** Home.md актуален, файл рефлексии создан и заполнен.

**Шаги:**

1. Прочитать `Home.md`
2. Если в блоке "Входящие" есть пункты — предложить решение по каждому: выбросить / переместить в нужный домен / оставить. Внести изменения.
3. Пройти по каждому домену из Home.md. Для каждого спросить: "Статус по Карьере — что сейчас актуально?" Обновить строки Статус и Фокус недели.
4. Спросить: "Какой главный фокус на эту неделю — одна вещь по всей жизни?"
5. Запустить операцию 2 (Рефлексия).

---

## Операция 2 — Рефлексия

**Результат:** файл `База знаний/Рефлексии/ГГГГ-ММ-ДД.md` создан и заполнен.

**Шаги:**

1. Прочитать шаблон `База знаний/Шаблоны/Шаблон еженедельной рефлексии.md`
2. Создать файл `База знаний/Рефлексии/[сегодняшняя дата].md` с заголовком `# Еженедельная рефлексия — [дата]`
3. Задавать вопросы из шаблона по одному — не все сразу. Ждать ответа на каждый перед следующим.
4. После каждого ответа записывать его в файл.
5. После раздела 7 — самостоятельно заполнить блок "Итоги" в начале файла рефлексии:
   - **Инсайты** — ключевые наблюдения о неделе, может быть несколько
   - **Проблемы** — хронические или острые проблемы которые всплыли
   - **Действия** — 2-3 конкретных шага, не размытых, с глаголом действия
6. Добавить новые итоги в блок "Итоги прошлой рефлексии" в `Home.md`:
   - Вставить новый блок СВЕРХУ существующих, не удалять старые
   - Формат: `_[[ГГГГ-ММ-ДД]]_` + Инсайты + Действия
   - Старые итоги остаются — владелец сам удаляет их по мере разбора
7. Обновить `База знаний/Рефлексии/Паттерны.md`:
   - Просмотреть проблемы и инсайты из текущей рефлексии
   - Если проблема уже есть в файле — добавить новую дату `_Повторилось: ГГГГ-ММ-ДД_`
   - Если проблема новая — добавить новый блок в "Активные паттерны" с датой
8. Показать итоги владельцу и сообщить что всё сохранено.

**Важно:** не торопить. Если ответ короткий — уточнить одним вопросом, не давить.

---

## Операция 3 — Новая база знаний

**Результат:** новый файл в `База знаний/[Домен]/knowledge/` создан по шаблону.

**Шаги:**

1. Спросить: "По какой теме создаём базу знаний?"
2. Определить домен (Карьера / Финансы / Здоровье). Если неочевидно — уточнить.
3. Прочитать `_meta/1. Шаблон для создания новой системы знаний.md`
4. Определить номер нового файла (посмотреть что уже есть в knowledge/).
5. Создать файл по шаблону.
6. Добавить ссылку на новый файл в обзорный файл домена со статусом ⚪.

---

## Операция 4 — Углубить базу знаний

**Результат:** выбранный knowledge-файл дополнен новым материалом.

**Шаги:**

1. Показать список файлов в `knowledge/` всех доменов. Попросить выбрать.
2. Прочитать `_meta/2. Инструкция для обновления и углубления существующей системы знаний.md`
3. Прочитать выбранный файл **целиком**.
4. Спросить: "Что добавляем — новая тема, уточнение, практика, конспект?"
5. Добавить материал в конец соответствующего раздела. Не переписывать существующее.
6. Обновить статус файла в обзоре домена если изменился.

---

## Операция 5 — Разобрать Входящие

**Результат:** блок "Входящие" в Home.md очищен, всё распределено.

**Шаги:**

1. Прочитать `Home.md`, извлечь все пункты из блока "Входящие".
2. Если пусто — сообщить "Входящие пусты, всё чисто."
3. По каждому пункту предложить один из вариантов:
   - Выбросить (не актуально)
   - Добавить в задачи домена (указать куда)
   - Оставить как мысль в блоке "Мысли" нужного домена
   - Создать новую базу знаний по теме (запустить операцию 3)
4. После решения по всем пунктам — очистить блок "Входящие" в Home.md.
