## Кто владелец и какие цели

Олег — frontend тимлид в IT. Параллельно осваивает крипто-трейдинг (скальпинг).

Главная цель: финансовая свобода и независимость через два канала — высокооплачиваемая IT-карьера и стабильный заработок на трейдинге. Подробнее — в [[Жизнь]].

---

## Что такое эта база знаний

Личная база знаний в Obsidian. Не энциклопедия — оперативный инструмент профессионала.
Принцип: не "что это такое", а "как устроено изнутри, как применять, где границы, какие риски".

## Структура

```
Home.md              — еженедельный дашборд. Точка входа для владельца

База знаний/
  01. Карьера/
    Карьера.md       — обзор домена: статус, задачи, мысли
    knowledge/       — базы знаний по разработке
  02. Финансы/
    Финансы.md       — обзор домена: статус, задачи, мысли
    knowledge/       — базы знаний по крипте и трейдингу
  03. Здоровье/
    Здоровье.md      — обзор домена: статус, задачи, мысли
    knowledge/       — тренировки, витамины, план по форме
  04. Жизнь/
    Жизнь.md         — мастер-документ: цели, ценности, вектор жизни
  Рефлексии/         — еженедельные рефлексии, формат ГГГГ-ММ-ДД.md
  Шаблоны/           — шаблоны для владельца (рефлексия и др.)
  Архив/             — завершённое и неактуальное

_meta/               — ты здесь. Всё техническое: контекст, инструкции, правила
```

## Правила работы с контентом

**Главное правило:** не теряй ни слова. Каждое слово в базе — личные конспекты и выводы владельца.

- Существующий контент не переписывать, не сокращать, не оптимизировать
- Можно только добавлять новое или перемещать файлы
- При создании новых баз знаний — использовать [[1. Шаблон для создания новой системы знаний]]
- При углублении существующих — использовать [[2. Инструкция для обновления и углубления существующей системы знаний]]

## Как работать с доменами

Каждый домен (`База знаний/01. Карьера/`, `База знаний/02. Финансы/` и т.д.) устроен одинаково:
- `[Домен].md` — обзорный файл: цель, статус, задачи, мысли, ссылки на knowledge
- `knowledge/` — файлы баз знаний по конкретным темам. Нумерация отражает порядок изучения

---

## Именование файлов

- Язык: русский
- Пробелы между словами
- Заглавная только первая буква
- Нумерация только внутри `knowledge/` для обозначения порядка

## Что делать в типовых ситуациях

### Создать базу знаний по новой теме
1. Прочитать [[1. Шаблон для создания новой системы знаний]] — следовать ему полностью
2. Определить в какой домен входит тема (`База знаний/01. Карьера/` или `База знаний/02. Финансы/` и т.д.)
3. Создать файл в `База знаний/[Домен]/knowledge/` с нумерацией по порядку
4. Добавить ссылку на новый файл в обзорный файл домена (`База знаний/[Домен]/[Домен].md`) со статусом ⚪

### Углубить или обновить существующую базу знаний
1. Прочитать [[2. Инструкция для обновления и углубления существующей системы знаний]] — следовать ему
2. Прочитать текущий файл целиком перед любыми изменениями
3. Только добавлять — не удалять и не переписывать существующее
4. Обновить статус файла в обзоре домена если изменился

### Создать новый домен
1. Создать папку `База знаний/0N. [Домен]/` и `База знаний/0N. [Домен]/knowledge/`
2. Создать обзорный файл `База знаний/0N. [Домен]/[Домен].md` по структуре: Цель / Где сейчас / Задачи / Мысли / База знаний
3. Добавить домен в `Home.md` и в `README.md`
4. Обновить этот файл (`_meta/README.md`) — раздел "Структура"

### Изменения в структуре базы
После любых структурных изменений — обновить `README.md` и при необходимости этот файл.
